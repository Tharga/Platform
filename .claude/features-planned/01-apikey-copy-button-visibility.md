# Feature Request: ApiKeyView — Copy key button always visible for unlocked keys

**Requested by:** Daniel Bohlin
**Date:** 2026-03-23

## Problem
In `ApiKeyView`, the "Copy key" button is only visible after pressing the "Show key" button. If a key is already unlocked/revealed, the user should be able to copy it without first clicking "Show key".

## Expected Behavior
The "Copy key" button should always be visible when there is an unlocked (revealed) key, not only after the "Show key" button has been pressed in the current session.

## Current Behavior
The "Copy key" button only appears after clicking "Show key", even if the key value is already available/unlocked.
